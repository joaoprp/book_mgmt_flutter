# book_mgmt

Pequeno projeto em flutter que gerencia livros

## Decisões e definições

Como essa é minha primeira experiência com Flutter, e com alguns percalços pessoais _justamente no dia do teste_, as decisões sobre o projeto foram focadas em ter algum fluxo apresentável. Quais são os fluxos:

1. Bloqueio de rotas por autenticação
2. Novo registro pela interface
3. Login
4. Navegação simples, listando livros e fazendo fuzzy search sobre os títulos dos livros
5. Habilidade de adicionar novo livro, sem índices

O projeto apesar de incompleto está em um ponto entregável, e requests diretas pela API são refletidas na interface (exceto índices).

## Pontos adicionais

Uma das coisas que eu senti falta (mas não tem a ver com o desafio e sim com a prática do dia-a-dia de trabalho) é de um design das interfaces. Eu não sou designer, e entre uma interface funcional e uma interface bonita há um designer de distância.

Dito isso, achei interessante o flutter e suas capacidades. Lembra bastante o react-native em um certo ponto (estrutura de objetos em objetos), mas tem uma fricção que o Typescript libera em async/await e typecasting.

A curva de entendimento e aprendizado do flutter é muito curta, pena que não tive todo o tempo do desafio para entregar ele inteiro. :/
